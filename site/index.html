How decisions are made

  1. Governance snapshots are exported from declared source pipelines.
  2. Only admitted resources from a public registry can be used.
  3. Public doctrine is applied through deterministic execution logic.
  4. Rationale + hashes + manifests are generated for replay.
  5. Artifacts are published for external audit.

What is explicitly disallowed

  • Private assistant memory or local journals
  • Private chats/DMs/unpublished notes
  • Hidden prompts or undeclared datasets
  • Manual score overrides outside public code and policy

Self-learning and self-correction policy

The system runs a policy-gated weekly review with hard-coded thresholds and only three outcomes: STAY_COURSE, TUNE_NON_FUNDAMENTAL, or PROPOSE_FUNDAMENTAL_CHANGE. Fundamental changes are never auto-applied.
